About Halton Digital Factory


Objective


Develop a website that integrates the entire manufacturing process, streamlining workflows and eliminating paper-based documentation.

Implementation Plan


Phase 1: Static Website on a Local Server


The transition to a Digital Factory starts by digitizing all engineering documents and establishing a centralized database for project parameters. The initial phase includes a static website hosted on a local server.

Phase 2: Dynamic Web Application on an HTTP Server


This phase introduces advanced search capabilities, allowing managers, sales teams, and engineers to retrieve historical project data efficiently. A role-based access control system will be implemented, ensuring appropriate access rights for users. All relevant data will be stored in an SQL database, facilitating job assignments and project tracking.

Phase 3: Expansion Across Additional Departments


Additional functionalities will be integrated to enable factory workers to log process start and end times. This will provide managers with real-time workflow insights and help analyze labor efficiency. Historical data will be leveraged to optimize project timelines and workforce planning.

Phase 4: AI Integration with DeepSeek


It is just an idea now, and I have no clear plan yet.

This journey may take years. We hope you’ll join us along the way and contribute ideas to further improve the system.

Current Status


We are currently in Phase 1 of the project.

Managing the Drawing Library


The website is designed for maintenance by engineers familiar with SolidWorks and my add-in application in SolidWorks. Engineering files are stored in the CustomizedProduct, StandardProduct, and StockPartLibrary subfolders.

CustomizedProduct Management


Customized product designs require frequent updates and daily maintenance. The manufacturing packages are saved in DigitalFactory/Content/CustomizedProduct.

Creating a Customized Project Manufacturing Package

1. Create a new project folder under the respective year (e.g., CustomizedProduct/ _CAN_24).

2. Use JustinTool to generate the manufacturing package to the folder.

3. The tool automatically updates the home page of the digital factory and the yearly project index file and integrates new BOM files.

4. Users can search and filter projects efficiently through the index file.

Additionally, project key parameters are stored in a JSON file for future data mining and search functionality enhancements.

StockPartLibrary Management


Adding a New Stock Part


1. Open StockPartHoodCommon.sldasm in SolidWorks.

2. Insert the new stock part assembly (e.g., RSB00.sldasm)

3. Export PDF, DXF, JPG files of stock parts (RSB01.sldprt, RSB02.sldprt) to the appropriate folder.

4. Save the HTML BOM file for the updated assembly in the corresponding directory.

5. The new stock parts will be reflected in StockPartHoodCommon.EASM and BOM-StockPartHoodCommon.html.

Creating a New Stock Part Subfolder


1. Create a new folder (e.g., KVE) under DigitalFactory/Content/StockPartLibrary.

2. Develop an assembly file (for example: StockPartKVE.sldasm) and set a custom property for notes.

3. Use JustinTool to generate a manufacturing package to the new folder, which will automatically update the stock part library.

StandardProduct Management


For products like control panels that undergo minimal design changes, manufacturing packages are stored in DigitalFactory/Content/StandardProduct.

Example: Creating a Standard Product Manufacturing Package


1. Create a job folder under CustomizedProduct (e.g., C25104A). Copy the following files into the folder: Submittal drawing: C25-104 - S1 (GL-L-38).dwg and Excel BOM for 38” A&W hood: BOM-GL-L-38C8104.5P.

2. Update Project Information in Excel BOM.

3. Launch JustinToolsBaske and locate the Excel BOM file.

4. Execute the Release Process by click “Add STD BOM For This Job”, then select the Standard Product Library Folder in the dialog.

5. Click Update Project Status to CNC or released.


Example: Adding a Standard Product to Library


1. Create a job folder under StandardProduct (e.g., StandardProduct\LCH\LCH2).

2. Open the LCH hood SolidWorks model.

3. Export BOM files, EASM, and PDF files to the folder (e.g., StandardProduct\LCH\LCH2).

4. The sub category of LCH index file will add the information for LCH2 automatically.

Maintaining the HTML Index File


Adding or Updating Project Information


1. Locate the HTML BOM file of the project.

2. If updating, uncheck the “Update All” option.

3. To delete a project from the index, check "Del From Index"; otherwise, the software will check for existing projects and update information as needed.

4. Click the appropriate update button based on the project’s location.

Updating the Index File for All Subfolders


1. Check the "Update All" option.

2. Click the corresponding button to update the subfolder index files. If the combo box displays "-- Select Subfolder --", it will update CustomizedProduct.html or StandardProduct.html.

System Requirements

To optimize search functionality and standardize data entry, the following requirements must be met:

1. Date Format: All dates should follow the YYYY-MM-DD format. Ensure Windows settings align accordingly.

2. Custom Property "Notes": A detailed description must be added as a custom property in SolidWorks assemblies. This will be extracted into HTML BOM files via JustinTool. Engineers can use Excel configurator to create the content of “notes” automatically.
